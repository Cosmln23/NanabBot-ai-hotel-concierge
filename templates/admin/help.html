LINE Connection Guide

  1. Go to LINE Developers Console
  2. Create a new Provider (or use existing)
  3. Create a new Messaging API channel
  4. Link to your LINE Official Account
You can create a free LINE Official Account at manager.line.biz

  1. Open your Messaging API channel
  2. Go to Basic settings tab, copy Channel Secret
  3. Go to Messaging API tab
  4. Under Channel access token section, click Issue
⚠️ Do NOT use short-lived tokens!

In LINE Developers Console → Messaging API tab → Channel access token: Click "Issue". Ensure you generate a LONG-LIVED token that does not expire. Do not use the 24-hour token, or your bot will stop working tomorrow!

After issuing the token, copy it immediately. You won't be able to see it again!

  1. Go to Integrations page
  2. Click Configure on Messaging Provider
  3. Select LINE from dropdown
  4. Paste your Channel Secret and Access Token
  5. Click Test Connection
  6. Copy the Webhook URL that appears
Go to Integrations

  1. Go back to LINE Developers Console
  2. Open Messaging API tab > Webhook settings
  3. Paste the Webhook URL from AI Hotel Suite
  4. Enable Use webhook
  5. Click Verify to test
Important: Disable auto-reply in LINE Official Account Manager to avoid duplicate messages.
Common Errors
Error Solution
Invalid token Re-issue token in LINE Developers Console
Webhook verification failed Check firewall, ensure HTTPS is configured
No messages received Enable "Use webhook" in LINE settings
Bot not responding Disable auto-reply in LINE Manager

WhatsApp Connection Guide

  1. Go to Meta Business Suite
  2. Create or use existing Business Account
  3. Complete business verification (may take 1-3 days)
Business verification requires official documents. Start this early!

  1. Go to Meta Developers
  2. Create a new App > Select Business type
  3. Add WhatsApp product to your app
  4. Note your Phone Number ID and WABA ID

  1. Go to Business Settings > System Users
  2. Create a System User with Admin role
  3. Assign WhatsApp assets to this user
  4. Generate token with whatsapp_business_messaging permission
Use a permanent token (System User), not the temporary test token!

  1. Go to Integrations page
  2. Click Configure on Messaging Provider
  3. Select WhatsApp (Custom)
  4. Enter Phone Number ID, Access Token, WABA ID
  5. Click Test Connection
  6. Copy Webhook URL and Verify Token
Go to Integrations

  1. Go to Meta Developers > WhatsApp > Configuration
  2. Click Edit next to Webhook
  3. Paste the Callback URL (Webhook URL from AI Hotel Suite)
  4. Paste the Verify Token
  5. Click Verify and save
  6. Subscribe to: messages, message_status
Common Errors
Error Solution
Invalid token Generate new token in Meta Developers
Phone number ID not found Copy exact ID from WhatsApp > Getting Started
403 Forbidden Ensure token has whatsapp_business_messaging permission
Messages not delivered First message to user must be template-based

Session Security

Prevent abuse from guests who already checked out:

  • QR Session Expiry - Sessions expire after 24-48 hours
  • Guests must re-scan QR to continue
  • PRO tier uses PMS checkout dates automatically

The bot auto-detects guest language. If it's responding incorrectly:

  1. Check "Staff Language" setting in your hotel profile
  2. Verify welcome message language matches your market
  3. For bilingual hotels, enable "Also send in English" option

Check the following:

  1. Verify "Food & Beverage" toggle is enabled in AI Settings
  2. Upload your menu with prices
  3. If product isn't in menu, bot will say it doesn't have info

Common QR issues:

  1. LINE: Guest must first add the bot as friend
  2. WhatsApp: Opens wa.me link - needs WhatsApp installed
  3. Re-download QR codes after changing messaging provider

If guests see "session expired":

  1. This is a security feature (QR Session Expiry)
  2. Guest must re-scan room QR to reconnect
  3. Adjust session hours (24h/48h) in AI Settings
  4. PRO tier: Uses PMS checkout date instead

When the bot can't help, guests can:

  1. Say "talk to human" or "speak to staff"
  2. A task will be created for your staff
  3. Staff can reply directly from the Tasks page
